Help for instructors

General principle of this exercising tool

This tool allows to train students to shell on a Unix system.

For students to be able to train by following exercises, they need to register to an exercise session.

A session is defined by a sequence of exercices, with a starting and an ending date, together with one or two more parameters indicating:

  • for each exercise, if a given score is needed to be reached to access the next exercise (set it to 0 for a free acceess to the next exercise, whatever the score to the current exercise.
  • if the session is timed, meaning that a new exercise is available every day.

An exercise sequence is an ordered list of exercises chosen among the available exercises on the platform. Each exercise sequence is associated with a profile (an objective with a description). As for now, a profile is associated to a job (the more common student's goal is to become job-ready) and to a skill level for that job(beginner, ..., expert).

Exercises

To fully grasp the following, instructors are advised to register to an existing session and try doing an exercise. Knowing how students solve exercises will indeed clarify a lot in the exercise authoring process.

All exercices follow the same operating mode:

  1. the student gets an (html formatted) statement containing instructions and questions) together with a question archive file to download
  2. the student downloads the archive and put on a unix system [1]
  3. the student extracts the archive content and works in the corresponding folder hierarchy following instruction guidelines
  4. the student creates an answer archive including the current content of the folder hierarchy that hopefully changed when following the exercise's instructions
  5. the student submits the answer archive by uploading it on the platform
  6. the platform analyzes the answer archive and indicates to the students good and bad things with respect to instructions that were given. The student also gets an overall success score for this submitted answer (from 0 to 100%).

The student can submit other answers to the exercise as much as he/she want (in the period during which the exercise is available -- see session details above). Every submitted answer is stored by the system, together with obtained feedback and score. Both the student and the instructor proposing the session can access details of the trials.

Possible actions for an instructor

Your usage of the platform will usually go from
  1. Creating exercises
  2. Creating a sequence of exercises
  3. Creating a session relying on a sequence
For the two first steps, you can re-use existing exercises and sequences (you have access to those authored by other instructors).

As an instructor you can:

  • create exercices, edit or delete exercises you authored
  • create/edit/delete exercise sequences.
  • create sessions and edit/delete those you created.
  • see results of students registered to your sessions

Warning: as an instructor, actions that you take on the platform can have important consequences for others. Please think twice before doing edits/deletes operations. For instance, other instructors might have decided to use an exercise you proposed in a session they planned. Exercise modifications will affect next students accessing the exercise. Currently exercise sequences are common to all instructors, so changing the order of the exerices will affect students of all sessions referring to this exercise sequence. We encourage instructors to include their name in the sequence name or description, to let other instructors know they're using the sequence.

In progress: we're currently working on a version where each instructor referring to an exercise of another instructor gets a distinct clone of the exercise, not to be affected by changes made by other authors. A similar idea will avoid authors to temper with sequences used by other authors.

Authoring a new exercise

Each exercise is composed of:

  • A template statement in HTML format containing generic instructions for students
  • A template question archive in .tar.gz format
  • A python script to generate individualized student statements and question archive from the templates [2]
  • A python script to analyze a student's answer archive [2]

To create an exercise, an instructor must provide all these elements. Though this can be done in several steps (an exercise status indicates whether the exercise is ready to be included in sessions).

Additionnally, the instructor can indicate skills associated to the exercise. Obtaining top scores to the exercise will allow students to earn the corresponding skill up to some level.

The simplest student statement creation scripts can simply provide a mere copy of the template archive and statement. But the power of such scripts lies in building different statements for different students, to provoke and promote discussions between peers and to avoid complete plagiarim of answers.

It is recommended that all the content of the question archive be contained in a single folder (possibily with many subfolders) to group the elements on which the students will work.

Concerning questions contained in an exercise statement, it is possible to ask students to write their answers in a file [such as 2] included in the question archive and to be completed by students. But, some questions just consist in changing the archive (reorganizing folders, creating file, modifying permissions, ...).

When a student submits an answer archive to an exercise, the python analyzing script is launched by the ShellOnYou platform and gets as input the HTML student statement (possiblly specific of the student), the question archive he/she was given at start (that can also be specific) and the answer archive the student just uploaded.

A python library is available to isntructors to reduce the production of boiler plate code. E.g., it provides current archive manipulation routines. The library is made available to both generating and analyzing scripts.

Moreover, a gitlab repository regroups the material for existing exercises (template archive, template statement, python scripts, answer archive examples). Instructors can access it by simple request at shellonyou AT polytech-reseau DOT org.

[1] Where can students solve a ShellOnYou exercise?

  • on their computer or device from a terminal (Linux, Free-BSD, WSL2 on Windows)
  • on a website from their browser: several sites provide free access to Unix (real or emulated) systems that allow files to be uploaded and downloaded (e.g., replit.com, CoCalc, ...). This is a little less comfortable but it allows them to work from the browser with nothing to change on their system when on a Windows OS.

[2] written in Python v2 (we plan to move to Python v3 depending on the demand)

[3] typically in an answers.txt file whose content is preformarted, and that is located at the root of the hierarchy in the question archive.
